Skin Definition for:

marvin

This is the skin definition file for my Euphoria Jabber Client. Note that while Marvin is in heavy development this information can and probably will change...

Resources:

scrollbar

Contains resources for the scrollbar control.

up

Image for the up button of the scrollbar control, should be the same size as down resource.

down

Image for the down button of the scrollbar control, should be the same size as up resource.

background

Image for the background of the scrollbar control, will be tiled.

thumb

Image for the thumb of the scrollbar.

roster

Contains resources for the roster.

group

Contains expanded and collapsed versions of the icon for group items on the roster.

expanded

Expanded version of the icon for group items on the roster.

collapsed

Collapsed version of the icon for group items on the roster.

item

Images for items (contacts) on the roster. Should be 16x16. If no images are supplied then no images will be used.

offline

Image for offline items (contacts) on the roster. Should be 16x16. If no image is supplied then no image will be used.

available

Image for available items (contacts) on the roster. Should be 16x16. If no image is supplied then no image will be used.

chat

Image for items (contacts) on the roster who are Free For Chat. Should be 16x16. If no image is supplied then the available image will be used.

away

Image for items (contacts) on the roster who are away. Should be 16x16. If no image is supplied then the available image will be used.

xa

Image for items (contacts) on the roster who are in extended away state. Should be 16x16. If no image is supplied then the away image will be used.

dnd

Image for items (contacts) on the roster who are in dnd state. Should be 16x16. If no image is supplied then the away image will be used.

systray

Images for the system tray icon. Will be resized to 16x16 with 16 colors.

available

Image to be shown in system tray when user's state is available. Will be resized to 16x16 with 16 colors.

chat

Image to be shown in system tray when user's state is free for chat. Will be resized to 16x16 with 16 colors.

away

Image to be shown in system tray when user's state is away. Will be resized to 16x16 with 16 colors.

xa

Image to be shown in system tray when user's state is extended away. Will be resized to 16x16 with 16 colors.

dnd

Image to be shown in system tray when user's state is do not disturb. Will be resized to 16x16 with 16 colors.

message

Image to be shown in system tray when a message is waiting. Will be resized to 16x16 with 16 colors.

Messages:


The following messages can be sent from any window, please see bellow for messages that can only be sent from specific windows.
Please note that not all of these messages make sense in all windows.
MessageRequiredDescription
LogOff true When the application recieves this message it will log the user off of the Jabber network, close the Roster window and show the Login window.
Settings true When the application recieves this message it will show the Settings window.
SetAvailable true When the application recieves this message it will set the users status to "Available".
SetChat true When the application recieves this message it will set the users status to "Free For Chat".
SetAway true When the application recieves this message it will set the users status to "Away".
SetXA true When the application recieves this message it will set the users status to "Extended Away".
SetDND true When the application recieves this message it will set the users status to "Do Not Disturb".
SetCustom true When the application recieves this message it will open up the Set Custom Status window.
ChangeSkin true When the application recieves this message it will show the Change Skin window.
AddContact true When the application recieves this message it will open up the add contact window.
Gateways true When the application recieves this message it will show the Gateway Settings window.
Find true When the application recieves this message it will show the Find Users window. NOT YET IMPLEMENTED
Menu true This message will display a popup menu containing all the functions required for the current window. The idea here is that if new features are added old skins will still be usable...
ChangePassword false This message will bring up a dialog that allows the user to change their password.
ToggleSounds false Will toggle sounds on or off.

Window: roster

This is the main window of the client, it shows a list of all contacts on the users roster and alows the user to send messages to them.

Areas:
AreaRequiredDescription
roster true The main part of the roster control will be placed in this area.

States:
NameDescription
rosterscroll true The scrollbar used to scroll the roster list will be places in this area.

States:
NameDescription
topmost false This area's state is changed to reflect whether the roster is currently a topmost window (hovers over other windows). I sugest that this area send the "topmost" message on left click, this will make it work like a toggle.

States:
NameDescription
on The image(s) to display when the roster is a topmost window
off The image(s) to display when the roster is not a topmost window

showstate false This area's state is changed to reflect the users current show state.

States:
NameDescription
available The image(s) to display when the users state is "Available" (normal)
chat The image(s) to display when the users state is "Free For Chat"
away The image(s) to display when the users state is "Away"
xa The image(s) to display when the users state is "Extended Away"
dnd The image(s) to display when the users state is "Do Not Disturb

Messages:
The following messages are valid within this window in addition to the global messages listed above.
MessageRequiredDescription
SendMessage false When the application recieves this message it will open up a message window to any users currently selected in the roster control. If no users are selected when this message is recieved it will have no effect.
RemoveContact true When the application recieves this message it will remove the selected contact from the roster (after asking the user for conformation).
Topmost false Toggles whether the window is topmost or not. See the "topmost" area/

Text Variables:
The following variuables are valid within this window in addition to the global variables listed above..
VariableDescription

$status$ This variable will changed to reflect the users status (a custom string that the user can change, is the same as show state by default)

Window: message

This window is used for both sending and recieving of messages. It contains a log control and a message control. Depending on user settings the message control may be used for viewing incoming messages and composing outgoing messages or just for composing messages. Also depending on user settings the log may show the log of a current chat or all the logs for a roster item.

Areas:
AreaRequiredDescription
log true The main part of the log control will be placed in this area. The log control displays either a log of the current chat or a complete log depending on user settings. It may also be the only place incoming messages are displayed (also depending on user settings).

States:
NameDescription
logscroll true The scrollbar control for scrolling the log control will be placed in this area.

States:
NameDescription
message true The main part of the message control will be placed in this area. The message control is used for composing outgoing messages and, depending on user settings, for displaying new incoming ones.

States:
NameDescription
messagescroll true The scrollbar control for scrolling the message control will be placed in this area.

States:
NameDescription
bold true This area's state will be changed to reflect the bold attribute of the currently selected text. I recomend that this area send the "bold" message on left click to make it work like a toggle.

States:
NameDescription
off The area will be set to use this state when it is off.
on The area will be set to use this state when it is on.

italic true This area's state will be changed to reflect the italic attribute of the currently selected text. I recomend that this area send the "italic" message on left click to make it work like a toggle.

States:
NameDescription
off The area will be set to use this state when it is off.
on The area will be set to use this state when it is on.

underline true This area's state will be changed to reflect the underline attribute of the currently selected text. I recomend that this area send the "underline" message on left click to make it work like a toggle.

States:
NameDescription
off The area will be set to use this state when it is off.
on The area will be set to use this state when it is on.

fontbutton true This area's state is changed when the font drop down is activated. I recomend that this area send the "font" message on left click.

States:
NameDescription
show The image attributes to use when the font drop down is droped down.
hide The image attributes to use when the font drop down is not droped down.

sizebutton true This area's state is changed when the size drop down is activated. I recomend that this area send the "size" message on left click.

States:
NameDescription
show The image attributes to use when the size drop down is droped down.
hide The image attributes to use when the size drop down is not droped down.

history true This area's state will be changed to reflect whether message history is shown or not. I recomend that this area send the "hisotry" message on left click to make it work like a toggle.

States:
NameDescription
show The area will be set to use this state for it's images when the history is shown.
hide The area will be set to use this state for it's images when the history is hidden.

recipientshowstate false This area's state is changed to reflect the recipients current show state.

States:
NameDescription
available The image(s) to display when the recipients state is "Available" (normal)
chat The image(s) to display when the recipients state is "Free For Chat"
away The image(s) to display when the recipients state is "Away"
xa The image(s) to display when the recipients state is "Extended Away"
dnd The image(s) to display when the recipients state is "Do Not Disturb

send/reply false This area's state is changed to reflect whether the message pane of the message window is being used to compose a message or view a message (see send/reply message below).

States:
NameDescription
send The area will be set to this state when the user is able to compose message (when there is not a message displayed in the message pane)
reply The area will be set ot this state when a message is displayed in the message pane.

messageevent false This area's state is changed when message events are recieved

States:
NameDescription
none The area will be set to this state when no message events are to be shown.
composing The area will be set ot this state when the remote user is composing a reply.

topmost false This area's state is changed to reflect whether the window is currently a topmost window (hovers over other windows). I sugest that this area send the "topmost" message on left click, this will make it work like a toggle.

States:
NameDescription
on The image(s) to display when the roster is a topmost window
off The image(s) to display when the roster is not a topmost window

Messages:
The following messages are valid within this window in addition to the global messages listed above.
MessageRequiredDescription
send/reply true This is a dual purpose message, when the user is composing a message it will cause that message to be sent. When a incoming message is displayed in the message control (this only happens when certain user settings are in force) it will cause the message to be removed so that the user can compose a reply.
font true Causes the font dropdown to drop down. See the "font" area.
size true Causes the font size dropdown to drop down. See the "size" area.
bold true Causes the bold attribute of currently selected text to be toggled. See the "bold" area.
italic true Causes the italic attribute of currently selected text to be toggled. See the "italic" area.
underline true Causes the underline attribute of currently selected text to be toggled. See the "underline" area.
history true Causes the display of message history to be toggled. See the "history" area.
searchlog true Brings up a search dialog that allows the user to search the log of their current conversation and the full message history (if history is currently shown)
Topmost false Toggles whether the window is topmost or not. See the "topmost" area.

Text Variables:
The following variuables are valid within this window in addition to the global variables listed above..
VariableDescription
$font$ This variable is set to the current font.
$size$ This variable is set to the current font size.
$recipientnick$ This variable is set to the Nick or JID (if no nick is set) of the recipient or the word "Multiple"
$recipientjid$ This variable is set to the JID of the recipient or the word "Multiple"
$recipientshowstate$ This variable will changed to one of the follwing to reflect the recipients current show state: Available, Free For Chat, Away, Extended Away or Do Not Disturb
$recipientstatus$ This variable will changed to reflect the recipients status (a custom string that a user can change, is the same as show state by default)
$send/reply$ This variable will be set to "Send" when the user is composing a message. If a incoming message is displayed in the message control (this only happens when certain user settings are in force) it will be changed to "Reply".
$messageevent$ This variable will be set to "User is replying" when a "composing" message event is recieved. At other times it will be set to nothing.
